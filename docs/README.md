## 🏗 Project Snapshot

- **Frontend:** React + TypeScript + Vite, TailwindCSS, Shadcn/ui
- **Backend:** Express (TypeScript)
- **Audio:** Web Audio API + FFT analysis
- **Build/Deploy:** Dockerfile, optional Azure Container Registry (ACR)

Key entry points:
- Client: `client/src/App.tsx`, `client/src/pages/Home.tsx`
- Server: `server/index.ts`, `server/routes.ts`
- Shared types: `shared/schema.ts`

## 🚀 How To Use These Docs

1. Start with `PROJECT_KNOWLEDGE_BASE.md` to understand architecture and flows.
2. When coding, follow `DEVELOPMENT_GUIDELINES.md` for style and quality.
3. Log notable work in `AURORA_LOG.md` (date, summary, changes, next steps).
4. For UI, consult `MVP_CSS_LIBRARIES.md` and component patterns in the client.

## 🔁 Updating the Docs

- Keep `AURORA_LOG.md` current for every meaningful change or decision.
- Update `PROJECT_KNOWLEDGE_BASE.md` when adding/removing core modules or flows.
- Reflect process changes in `DEVELOPMENT_GUIDELINES.md`.
